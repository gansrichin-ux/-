# Logist App - Enhanced Logistics Management System

Улучшенное логистическое приложение с полным функционалом для управления грузами, водителями и клиентами.

## 🚀 Новые возможности

### ✅ State Management (Riverpod)
- Реализован современный подход к управлению состоянием с помощью Riverpod
- Централизованные провайдеры для всех сущностей (грузы, пользователи, геолокация)
- Оптимизированная производительность и кэширование данных

### 🔍 Поиск и фильтрация
- Полнофункциональный поиск по грузам с фильтрацией по статусам
- Мгновенный поиск по названию, маршруту и водителю
- Умная фильтрация с сохранением состояния

### 🔄 Pull-to-refresh
- Обновление данных свайпом вниз во всех списках
- Визуальная индикация процесса обновления
- Автоматическая синхронизация с сервером

### 📊 Дашборд со статистикой
- Детальная статистика по грузам и пользователям
- Интерактивные графики и диаграммы
- Показатели эффективности в реальном времени

### 📍 GPS трекинг
- Отслеживание местоположения водителей в реальном времени
- Автоматическое обновление координат
- Управление разрешениями на геолокацию

### 🗺️ Навигация и карты
- Интерактивные карты с отображением маршрутов
- Построение оптимальных маршрутов доставки
- Интеграция с внешними картографическими сервисами

### 📸 Фотоотчеты и подписи
- Создание фотоотчетов о доставке
- Сбор подписей клиентов
- Загрузка и хранение фотографий в облаке

### 👥 Управление клиентами
- Полная база данных клиентов
- История заказов и статистика
- Управление контактной информацией

### 📄 Система документов
- Загрузка и хранение документов
- Поддержка различных форматов
- Система тегов и категоризации

### ✅ Валидация форм
- Комплексная валидация всех форм ввода
- Пользовательские сообщения об ошибках
- Защита от некорректных данных

### 📱 Офлайн-режим
- Кэширование данных для работы без интернета
- Синхронизация при восстановлении соединения
- Очередь ожидающих действий

### 🧪 Unit тесты
- Покрытие ключевых компонентов тестами
- Тесты валидации и обработки ошибок
- Автоматизированное тестирование

## 🏗️ Архитектура

### SOLID принципы
- **Single Responsibility**: Каждый класс имеет одну ответственность
- **Open/Closed**: Классы открыты для расширения, закрыты для изменения
- **Liskov Substitution**: Подклассы могут заменять базовые классы
- **Interface Segregation**: Мелкие, специфичные интерфейсы
- **Dependency Inversion**: Зависимости от абстракций, не от конкретных реализаций


## 🚀 Запуск приложения

1. Установите зависимости:
```bash
flutter pub get
```

2. Настройте Firebase:
   - Создайте проект в Firebase Console
   - Добавьте файлы конфигурации (android/app/google-services.json, ios/Runner/GoogleService-Info.plist)
   - Включите Authentication, Firestore, Storage

3. Запустите приложение:
```bash
flutter run
```

## 🧪 Тестирование

Запуск всех тестов:
```bash
flutter test
```

Запуск конкретного теста:

```bash
flutter test test/unit/validation_test.dart
```

Генерация mock-классов:
